Introduction

Introduction

Why write another React guide? Early in my development career I had a hard time learning and grasping the concepts of ASP.NET. The learning curve felt steeper than Mount Everest. In part, .NET is a large ecosystem with much to offer but I believe some of the difficulty stemmed from "beginner" tutorials that assumed too much of readers. Tutorials that took for granted templated projects and scaffolded boilerplate code. Tutorials that glossed over IDE features and did little to explain the startup process or lifecycle of a running application.

The world of JavaScript has grown into a complex one and a close friend of mine is having a hard time navigating the ever-changing landscape. Who can blame them. JavaScript use to be a collection of script tags containing nothing more than jQuery DOM manipulations located in the head or body of an html page. Now a days, large JS applications may contain hundreds of files, often involve a build process, external dependencies from various locations, and must deal with problems that stem from asynchronous programming.

JavaScript is no longer just a language but a complex ecosystem consisting of runtime environments, package managers and build tools. The language itself comes in a variety of flavors with varying support. More on that later. If that is not enough, developers may even forgo JavaScript altogether and write in a language that compiles to JavaScript. Again, more on this later.

My goal for this book is to start with nothing and to build everything. To start from a basic React example with a simple build process and, like much of development, take an iterative approach to building out a feature-filled React application with a robust build process. To slowly build a knowledge base of React, JavaScript and the surrounding ecosystem. Above all else, my goal is to not only show code but explain the reasoning of how said code came to be.
